# List SW Dev File Trees

Regenerates the canonical ranked list of the **25 most-used file tree structures
in software development** and writes it to `README.md`, then records what changed
in `CHANGELOG.md`.

## When to activate

- Bi-weekly automated run via GitHub Actions (`.github/workflows/update-trees.yml`)
- Manual trigger: "refresh the file tree list", "update README", "regenerate trees"

## Output contract

| File | Action |
|------|--------|
| `README.md` | **Overwrite** with fresh ranked table (see format below) |
| `CHANGELOG.md` | **Append** one dated entry describing what changed |

## Step-by-step instructions

### 1. Research & rank

Mentally survey current developer ecosystem usage signals:
- Package manager defaults (npm, cargo, pip/uv, maven, gradle)
- GitHub repository conventions and template repos
- Framework-mandated layouts (Next.js, Django, Rails, Flutter, Spring Boot)
- Architecture-driven layouts (Clean, Hexagonal, Feature-Sliced)
- Tooling layouts (Terraform, Podman, Docker, CI/CD, Data Science)
- Agent Skills Layout (agentskills.io)
- Emerging patterns since the last run

Rank by **breadth of adoption across languages, ecosystems and developer surveys**.
Most widely used = rank #1.

### 2. Write README.md

Use exactly this structure — no extra headings, no prose before the table:

```markdown
# 25 Most Used File Trees in Software Development

> Ordered by prevalence of use across the software development ecosystem.
> Last updated: YYYY-MM-DD

| # | Name | Summary Description | Detail |
|---|------|---------------------|--------|
| 1 | **Name** | One-line summary | Two sentences of detail. Second sentence adds context. |
...
| 25 | ... | ... | ... |

---

*Generated by [list-sw-dev-file-trees-en](skills/list-sw-dev-file-trees-en/SKILL.md)
skill — runs bi-weekly via GitHub Actions.*
```

Rules:
- Exactly 25 rows, numbered 1–25
- **Name** column: bold, concise canonical name (e.g. `src/ + tests/`, `Maven Standard Directory Layout`)
- **Summary Description** column: ≤ 12 words
- **Detail** column: exactly 2 sentences; sentence 1 = what it contains, sentence 2 = adoption context
- Update the `Last updated:` date to today

### 3. Append CHANGELOG.md

Add a new entry at the **top** of the file (newest first):

```markdown
## YYYY-MM-DD

- Re-ranked: [describe any rank changes, e.g. "Monorepo moved from #8 to #6"]
- Added: [any new entry that replaced an older one]
- Removed: [any entry dropped]
- No change: [if list is identical to prior run, state "No ranking changes this run"]
```

### 4. Verify before finishing

- [ ] README.md has exactly 25 table rows
- [ ] `Last updated:` date is today
- [ ] CHANGELOG.md new entry is at the top
- [ ] No broken Markdown (no unclosed fences, no missing pipe characters)
